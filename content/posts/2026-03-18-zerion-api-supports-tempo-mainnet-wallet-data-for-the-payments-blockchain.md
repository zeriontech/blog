<p><a href="https://tempo.xyz/"><u>Tempo</u></a> mainnet is live! The blockchain built for real-world payments, incubated by Paradigm and Stripe, now runs in production with instant settlement, predictable low fees, and high throughput.</p><p>Zerion supported Tempo from its <a href="https://zerion.io/blog/zerion-supports-tempo-the-payment-first-blockchain/"><u>testnet launch</u></a>. With mainnet, that support moves to production. Developers building payment processors, wallets, or financial tools on Tempo can access clean, indexed wallet data through Zerion API without running their own infrastructure.</p><h2 id="what-zerion-covers-on-tempo">What Zerion covers on Tempo</h2><p>Zerion API provides the same endpoints on Tempo that developers already use across EVM chains and Solana:</p><ul><li><a href="https://developers.zerion.io/reference/listwalletpositions"><strong><u>Positions</u></strong></a><strong>:</strong> Token balances for any Tempo address, returned in sub-seconds.</li><li><a href="https://developers.zerion.io/reference/getwalletchart"><strong><u>Portfolio balance</u></strong></a><strong>:</strong> Portfolio value over time, ready for charts and dashboards.</li><li><a href="https://developers.zerion.io/reference/listwallettransactions"><strong><u>Transactions</u></strong></a><strong>:</strong> Parsed, human-readable transaction history for reporting, reconciliation, and tax compliance.</li><li><a href="https://developers.zerion.io/reference/findwallettransactionssubscription"><strong><u>Webhooks:</u></strong></a> Subscribe to new transactions on any Tempo address for real-time notifications and event-driven workflows.</li></ul><p>All of this uses Zerion's unified schema and a consistent data format across 38+ chains. If you already integrate Zerion API for Ethereum or Solana, adding Tempo takes minimal code changes.</p><h2 id="why-this-matters-for-tempo-builders">Why this matters for Tempo builders</h2><p>Tempo's architecture creates specific data challenges.&nbsp;</p><p>The chain has no native token. Fees are paid directly in stablecoins. It’s very fast: supports 100,000+ TPS with sub-second finality. Running your own indexers at that throughput is expensive and complex.</p><p>Zerion API abstracts this away. You get normalized, enriched wallet data through a single REST call, regardless of how many transactions the chain processes. Data comes fully interpreted, enriched, and ready to plug into interfaces.&nbsp;</p><p>For teams building payment apps, this covers common needs out of the box: tracking stablecoin balances across addresses, pulling transaction history for ledger reconciliation, and computing portfolio values in fiat. Tempo's stablecoin-native gas model makes accurate balance tracking especially important, because your users' fee tokens and payment tokens are the same assets.</p><h2 id="get-started">Get started</h2><p>If you're building on Tempo, you can start pulling wallet data today.</p><ol><li><a href="https://dashboard.zerion.io/"><u>Create a free Zerion API key</u></a></li><li>Call the positions endpoint for any Tempo address</li><li>Check the <a href="https://developers.zerion.io/"><u>API docs</u></a> for the full reference</li></ol><p>For custom RPS tiers, dedicated support, SLAs, and other enterprise needs, <a href="https://zerion.io/api#email-form" rel="noreferrer"><u>reach out to the team</u></a>.</p>
