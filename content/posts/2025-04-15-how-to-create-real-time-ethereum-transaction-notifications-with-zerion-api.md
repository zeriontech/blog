<p>Real-time transaction alerts are crucial for developers building Web3 apps, wallets, or portfolio trackers. With <a href="https://zerion.io/api" rel="noreferrer">Zerion API</a>'s webhooks, you can easily subscribe to transactions for Ethereum and all major EVM blockchains. In this blog post, we’ll explore how to quickly set up your webhooks to create alerts or notifications for transactions.&nbsp;</p><h2 id="why-use-transaction-webhooks">Why Use Transaction Webhooks?</h2><p>Imagine a scenario when you want to follow transactions of a specific Ethereum address in real-time.&nbsp;</p><p>You might want to copy-trade someone. Trigger an offchain action. Or send a push notification to the user who did the transaction, like in Zerion on mobile.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://zerion.io/blog/content/images/2025/04/Notifications.png" class="kg-image" alt="Screenshot of push notifications for transactions in Zerion" loading="lazy" width="1899" height="1096" srcset="https://zerion.io/blog/content/images/size/w600/2025/04/Notifications.png 600w, https://zerion.io/blog/content/images/size/w1000/2025/04/Notifications.png 1000w, https://zerion.io/blog/content/images/size/w1600/2025/04/Notifications.png 1600w, https://zerion.io/blog/content/images/2025/04/Notifications.png 1899w" sizes="(min-width: 720px) 720px"><figcaption><span style="white-space: pre-wrap;">An example of push notifications for transactions in Zerion</span></figcaption></figure><p>One way to do this is to regularly poll the blockchain to see if the address has made any new transactions.&nbsp;</p><p>However, this approach can quickly become inefficient, especially if you're monitoring multiple addresses across several chains. It would also require you to run 24/7 infrastructure to make sure you don’t miss any transactions.</p><p><a href="https://zerion.io/api"><u>Zerion API</u></a> does this heavy lifting and offers an easy way: webhooks with <a href="https://developers.zerion.io/reference/createsubscriptionwallettransactions#/"><u>subscriptions to transactions</u></a>.&nbsp;</p><p><strong>Webhooks</strong> allow your application to receive instant notifications whenever specified addresses make transactions, without continuously polling blockchain data. This helps developers:</p><ul><li>Update user balances immediately</li><li>Trigger real-time alerts and events</li><li>Enhance user engagement with timely notifications</li></ul><p>Webhooks are what power mobile notifications for addresses in Zerion Wallet. Now let’s see how easy it is to set them up.</p><h2 id="getting-started-with-zerion-transaction-webhooks">Getting Started with Zerion Transaction Webhooks</h2><p>Zerion API provides a straightforward approach to subscribing to transactions made by EVM addresses on Ethereum and all major EVM L1 and L2 chains.</p><h3 id="step-1-create-a-subscription">Step 1: Create a Subscription</h3><p>To subscribe to transactions, send a request to Zerion API, specifying:&nbsp;</p><ul><li><strong>Addresses</strong> you want to subscribe to&nbsp;</li><li><strong>Callback URL</strong>, where you’ll receive POST — <a href="https://webhook.site/" rel="noreferrer">webhook.site</a> is supported right away and custom URLs need to be approved&nbsp;</li><li><strong>Chain IDs </strong>where you want to get or leave it empty to subscribe to all supported blockchains</li></ul><p>Here is an example of a request:&nbsp;</p><pre><code class="language-bash">curl --request POST \
     --url https://api.zerion.io/v1/tx-subscriptions \
     --header 'accept: application/json' \
     --header 'authorization: Basic emtfZGV2X2Q2YmEzYzM3NTNiYjQ2NWM4YjNjZTEzMTU2OTVlYjMwOg==' \
     --header 'content-type: application/json' \
     --data '
{
  "addresses": [
    "0x42b9dF65B219B3dD36FF330A4dD8f327A6Ada990"
  ],
  "callback_url": "https://webhook.site/dd3c5376-2472-4b3f-af38-d75cc20ad1a2",
  "chain_ids": [
    "base"
  ]
}
'
</code></pre>
<h3 id="step-2-handling-webhook-notifications">Step 2: Handling Webhook Notifications</h3><p>Once subscribed, Zerion will POST transaction details directly to your webhook URL whenever the monitored address makes transactions.</p><p>If you used webhook.site, you will start seeing POST requests with full information for each transaction. </p><figure class="kg-card kg-image-card kg-width-wide kg-card-hascaption"><img src="https://zerion.io/blog/content/images/2025/04/Webhooks--1-.png" class="kg-image" alt="Transaction data immediately shows up as POST in webhook.site" loading="lazy" width="1500" height="1096" srcset="https://zerion.io/blog/content/images/size/w600/2025/04/Webhooks--1-.png 600w, https://zerion.io/blog/content/images/size/w1000/2025/04/Webhooks--1-.png 1000w, https://zerion.io/blog/content/images/2025/04/Webhooks--1-.png 1500w" sizes="(min-width: 1200px) 1200px"><figcaption><span style="white-space: pre-wrap;">Transaction data immediately shows up as POST in webhook.site</span></figcaption></figure><p>Each request will have the full information about the transaction. The exceptions are <code>price</code> and <code>value</code>, these will be <code>null</code>. Those come with a slight delay and need to be fetched by querying <a href="https://developers.zerion.io/reference/listwallettransactions#/"><u>Zerion API’s transaction endpoint</u></a> by hash.&nbsp;&nbsp;</p><p>Your webhook handler can then parse this data and take immediate action—updating a database, sending a notification, or triggering other events.</p><h2 All you need to do is define the address you want to watch and a URL (e.g. on webhook.site) where you want to get transactions.
