<h4 id="the-easiest-way-to-integrate-defi-protocols">The easiest way to integrate DeFi protocols</h4><p>Zerion has been working hard to stay up to date with innovations in the DeFi space. This means integrating as many protocols as possible, allowing users to track and manage their entire DeFi portfolio from one place. However, adding new protocols is nowhere near efficient. Our team has spent months designing a solution that makes this process significantly easier. Today, we’re excited to announce <strong>DeFi SDK</strong>, an open-source system of smart contracts that makes it easier for developers to integrate DeFi protocols.</p><h3 id="building-the-bridges-that-defi-needs">Building the bridges that DeFi needs</h3><p>Over the past year, we added support for numerous protocols including Uniswap, MakerDAO CDPs and Vaults, Compound, Bancor, 0x, Uniswap Pools, DSR, Fulcrum and Tokensets. The DeFi ecosystem is growing at an unprecedented speed. This is exciting and challenging at the same time. More and more financial products that were previously available to only a few are now globally accessible. <strong>As a company, we recognize that in order to keep up with this growth, we need to scale the way we approach integrations with DeFi protocols.</strong></p><figure class="kg-card kg-embed-card"><blockquote class="twitter-tweet"><a href="https://twitter.com/zerion_io/status/1234889021475770369"></a></blockquote><script async="" src="https://platform.twitter.com/widgets.js" charset="utf-8"></script></figure><p>In the past, we’ve spent a lot of time researching the ins and outs of each protocol we want to work with — thinking through the UX and how it fits our interface and developing the backend and frontend for all our clients on web, iOS and Android. The pipeline has often been fragmented and time-consuming.</p><p>DeFi SDK changes everything, not just for Zerion, but for all dApp and wallet developers. The best part? It’s trustless and completely <a href="https://github.com/zeriontech/defi-sdk" rel="noopener">open-source</a>.</p><p>We’ve designed DeFi SDK with three goals in mind:</p><ol><li><strong>For developers:</strong> If you’re a dApp developer, integrating DeFi protocols is as easy as using a single library. DeFi SDK gives you instant access to 17+ leading DeFi protocols instead of manually integrating each of them. If you’re a protocol developer, any updates you make are automatically reflected on all dApps that use DeFi SDK.</li><li><strong>For the community:</strong> We’re sharing open-source solutions to common problems. DeFi SDK increases interoperability of different DeFi products and make it easier to build DeFi apps</li><li><strong>For users:</strong> dApps can now support your favourite DeFi assets and protocols with ease. This means faster updates and a better user experience.</li></ol><h3 id="how-defi-sdk-works">How DeFi SDK works</h3><p>At the core of DeFi SDK are <strong>token adapters</strong> and <strong>protocol adapters</strong>.</p><h4 id="token-adapters"><strong>Token Adapters</strong></h4><p>Token adapters are smart contracts that provide an on-chain solution to fetch the underlying tokens of derivative assets. Think of these as decoders that make sense of complex ERC20 tokens.</p><figure class="kg-card kg-image-card"><img src="https://zerion.io/blog/content/images/max/800/0-niUje5nzLyM6eLzx.png" class="kg-image" alt="Zerion DeFi SDK Token Adapter Uniswap ETH-cDAI pool" loading="lazy" width="800" height="400" srcset="https://zerion.io/blog/content/images/size/w600/max/800/0-niUje5nzLyM6eLzx.png 600w, https://zerion.io/blog/content/images/max/800/0-niUje5nzLyM6eLzx.png 800w" sizes="(min-width: 720px) 720px"></figure><p>Imagine a Uniswap ETH-cDAI Pool token. On the surface, it is a very complicated token, but under the hood, it just consists of a pair of yield-earning DAI (cDAI) and ETH. The token adapter for UniTokens is able to “unwrap” the token and return the underlying components of the derivative token. Similarly, an adapter for cTokens is able to fetch the underlying interest-earning token. You can take a look at the Uniswap adapter on our Github.</p><h4 id="protocol-adapters"><strong>Protocol Adapters</strong></h4><p>Protocol adapters are responsible for the on-chain representation of a DeFi protocol and include user-specific information such as debt and deposit balance.</p><p>Currently, DeFi faces a fragmentation problem because wallets like Fortmatic or MetaMask can fetch balances of basic ERC20 tokens, but have no standard for reading more complicated DeFi derivatives such as Set tokens. Rather than query user balances on-chain, this information often needs to be hardcoded or pulled through API integrations. This is not ideal.</p><figure class="kg-card kg-image-card"><img src="https://zerion.io/blog/content/images/max/800/0-vxeERaV9MpBUMZJV.png" class="kg-image" alt="Zerion DeFi SDK Protocol Adapter Synthetix Aave PoolTogether" loading="lazy" width="800" height="400" srcset="https://zerion.io/blog/content/images/size/w600/max/800/0-vxeERaV9MpBUMZJV.png 600w, https://zerion.io/blog/content/images/max/800/0-vxeERaV9MpBUMZJV.png 800w" sizes="(min-width: 720px) 720px"></figure><p>DeFi SDK contains metadata for each protocol. For dApp developers, this allows the interface to fetch all information related to the protocol using one call to the <strong>Adapter Registry</strong>, which is a whitelist of the protocol adapters and token adapters supported by each interface.struct ProtocolMetadata {    <br>    string name;    <br>    string description;    <br>    string websiteURL;    <br>    string iconURL;    <br>    uint256 version;<br>}</p><p>Anyone can create a token adapter for their protocol and anyone can maintain their own registry for their wallet or dApp. For example, Zerion’s Adapter Registry is a vetted list that contains all the protocols that we support. You can find the addresses of deployed contracts, including the Adapter Registry <strong>here</strong>.</p><p>For protocol developers: adding a new protocol to DeFi SDK</p><p>The advantage to protocols that add an Adapter to DeFi SDK is that once integrated, their protocol data is accessible to every wallet and dApp that has DeFi SDK integration. When you update the protocol, these changes are automatically reflected in the dApps.</p><p>Applications currently using DeFi SDK are:</p><ul><li><a href="https://app.zerion.io/" rel="noopener">Zerion</a></li><li>MyDeFi</li><li>Frontier</li><li>DeFi Market Cap</li></ul><p>So, you have a new protocol that you think DeFi users will love — now what? Just follow these simple steps:</p><ol><li>Check out our open-source examples of Adapters <a href="https://github.com/zeriontech/defi-sdk/tree/master/contracts/adapters" rel="noopener">here</a>.</li><li>Create your own adapter smart contract, deploy it on the <a href="https://zerion.io/ethereum-wallet" rel="noreferrer">Ethereum</a> mainnet and verify it on Etherscan. (If you have any problems, reach out to us on Discord and we will be more than happy to help you!)</li><li>Once the adapter is complete, open a pull request on Github, we will need to review it and add it to the Zerion AdapterRegistry. Check out Github for more info on this.</li></ol><p>Once this process is complete, your protocol will be almost instantly visible to the users of Zerion and other dApps that use DeFi SDK.</p><h3 id="for-wallet-and-dapp-developers-how-to-integrate-defi-sdk-in-your-app">For wallet and dApp developers: How to integrate DeFi SDK in your app</h3><p>Wallet and dApp developers can make use of the Adapter Registry as a way to keep track of all DeFi assets for their users. Standardizing the way complex derivatives get integrated across wallets and protocols means that you can easily fetch key information such as user balance, protocol metadata, derivative token metadata, etc. Doing this is as easy as running a single function on the Registry:function getBalances(address account) returns (ProtocolBalance[]);</p><p>For each protocol listed on the Registry, this function returns all balances of a user across all tokens listed in the protocol adapter.</p><p>The advantage of integrating DeFi SDK just once is that your app will automatically support all existing and future protocols added to DeFi SDK.</p><h3 id="%F0%9F%9A%80-so-what%E2%80%99s-next-for-defi-sdk">🚀 So what’s next for DeFi SDK?</h3><p>This first version of DeFi SDK is for precise DeFi accounting. Our next step is to introduce Interactive Adapters that standardize the way dApps interact with multiple DeFi protocols. We are incredibly excited to work with developers, users and the wider DeFi community to make these integrations as secure and accessible as possible.</p><blockquote><strong>DeFi SDK solves the fragmentation problem across the DeFi space while staying trustless and open-source.</strong></blockquote><blockquote><strong>The way DeFi should be.</strong></blockquote><hr><p>Give us a ⭐️ on Github!</p><p>🏦 Follow us on Twitter to get latest updates and join our Discord server!</p>
