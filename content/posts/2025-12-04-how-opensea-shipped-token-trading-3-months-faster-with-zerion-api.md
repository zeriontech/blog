<p>When building at the scale of OpenSea, speed and coverage are everything. Launching token&nbsp;trading across multiple chains could mean heavy indexing work and infrastructure management. Instead, OpenSea tapped into Zerion API to accelerate its go-to-market and unlock new possibilities for traders.</p><div class="kg-card kg-callout-card kg-callout-card-blue"><div class="kg-callout-text"><b><strong style="white-space: pre-wrap;">TL;DR: </strong></b>OpenSea integrated the Zerion API to enable ERC-20 token trading on OS2, its fully rebuilt platform. With Zerion’s Wallet Positions endpoint, they shipped 3+ months faster, covering all chains from day one, and eliminating the need for custom indexing. </div></div><h2 id="the-challenge-multichain-token-data-at-scale">The Challenge: Multichain Token Data at Scale</h2><p>As the world’s largest NFT marketplace, OpenSea already operates at a massive scale.&nbsp;</p><p>When OpenSea announced <a href="https://opensea.io/blog/articles/introducing-os2"><u>OS2</u></a>, it wasn’t just a facelift, it was a rebuild from the ground up. The launch of OS2 brought NFTs and token trading together in one platform, expanded support for new chains, and introduced features like cross-chain purchases and real-time portfolio insights.</p><p>Expanding into ERC-20 token swaps put scale and development velocity into focus:</p><ul><li><strong>Time to market</strong> — Indexing and normalizing token data across OpenSea’s 18+ <a href="https://support.opensea.io/en/articles/8867082-which-blockchains-are-compatible-with-opensea"><u>chains supported</u></a> at the time would require significant engineering effort.</li><li><strong>Scale requirements</strong> — Any solution needed to support tens of millions of requests daily while staying performant.</li></ul><p>Launching OS2 meant evolving OpenSea into a truly multichain, multi-asset platform. This required solving speed, scale, and complexity simultaneously, which was a serious challenge even for the world’s largest NFT marketplace.</p><h2 id="the-solution-zerion-api%E2%80%99s-wallet-positions-endpoint">The Solution: Zerion API’s Wallet Positions Endpoint</h2><p>To accelerate development, OpenSea integrated Zerion API’s<a href="https://developers.zerion.io/reference/listwalletpositions#/"> <u>Wallet Positions endpoint</u></a>.</p><p>With a single call, Zerion’s endpoint returns all normalized token positions for any wallet across 26+ chains, including the 18 EVM networks supported by OpenSea. Balances are enriched with metadata and valued in USD, allowing developers to skip indexing entirely and focus on the product.</p><p>When selecting the API provider, several key advantages stood out for OpenSea:</p><ul><li><strong>18 chains covered from day one </strong>— Zerion supported all chains that OpenSea needed for the release, eliminating the need to integrate different solutions for different networks&nbsp;</li><li><strong>Normalized balances in one response — </strong>A single API call returned token balances for all supported chains</li><li><strong>Battle-tested infrastructure</strong> — Zerion already handled data at a consumer scale, both for its own products and for other API partners.</li></ul><h2 id="the-results-shipping-3-months-faster-467k-dau-supported">The Results: Shipping 3+ Months Faster, 467k DAU Supported</h2><p>By integrating Zerion API for token trading in OS2 since day one, OpenSea unlocked:</p><ul><li><strong>Faster time to market — </strong>The OpenSea team shipped token trading 3 months faster than an internal build, saving significant engineering resources and accelerating OS2’s roadmap.</li><li><strong>Full chain coverage with less complexity — </strong>All 18 supported chains integrated through a single API, cutting operational overhead.</li><li><strong>Reliability at scale — </strong>Zerion API served up to <a href="https://www.theblock.co/post/357409/opensea-hits-highest-monthly-users-since-2023-as-os2-exits-beta-trading-volume-remains-well-below-previous-highs"><u>467,322 daily active users of OS2</u></a>, handling up to 1,000 requests per second.</li></ul><p>With token trading live, OpenSea has now been able to deepen the trader experience. As the next step, OpenSea integrated <a href="https://developers.zerion.io/reference/getwalletpnl"><u>Zerion API’s PnL endpoint</u></a> to help traders understand their performance across chains and protocols. <a href="https://support.opensea.io/en/articles/12417473-how-can-i-view-my-pnl"><u>OpenSea’s estimated profit &amp; loss</u></a> shows the amount invested, realized, unrealized, and return. </p><blockquote>“Zerion's API got us to market 3 months faster than an in-house build (an eternity in crypto) and with enough flex capacity to support 18 chains through switch-on of one of the largest rewards programs in the industry. We continue to see Zerion as a long-term partner in speed-to-market, and one of few providers in web3 today that offer APIs that scale to OpenSea's userbase.” — Chris Maddern, CTO at OpenSea</blockquote><h2 id="build-like-opensea">Build Like OpenSea</h2><p>OpenSea’s OS2 launch included a serious challenge: unify token data across 18 chains, handling hundreds of millions of real-time requests.
