<p>AI agents are reshaping the way we interact with crypto. But to build automation that works in the real world, you need fast, reliable access to onchain data. That’s exactly the challenge Loomlay faced as they grew their no-code agent platform from informational bots to advanced, automated trading strategies.</p><p>In this blog post, you’ll see how <a href="https://loomlay.com/" rel="noreferrer">Loomlay</a> integrated Zerion API to give AI agents real-time wallet and transaction data. This unlocked new capabilities while cutting development time.&nbsp;</p><h2 id="what-is-loomlay">What is Loomlay?</h2><p>Loomlay is a no-code platform that empowers users to build and deploy AI-powered agents for crypto and DeFi applications.&nbsp;</p><p>With Loomlay, anyone can create custom agents without writing a single line of code. Agents’ capabilities can range from shitposting on X to automating advanced strategies. Over time, Loomlay has evolved from building simple information agents to launching a full-fledged trading agent store, enabling copy trading, DCA (dollar-cost averaging), and more complex automated strategies.</p><p>Loomlay’s vision is to make crypto automation accessible, flexible, and powerful for everyone, from retail traders to power users. This means supporting various data sources and integrations across the DeFi landscape.</p><h2 id="the-challenge-accessing-wallet-data-fast-and-at-scale">The Challenge: Accessing Wallet Data, Fast and at Scale</h2><p>As Loomlay expanded from informational social media agents to trading automation, they needed a reliable, up-to-date onchain wallet data.&nbsp;</p><p>Trading and portfolio management agents must know what assets are held by a wallet address. They also need to track transaction histories in real time. This portfolio and transactions data is required for Ethereum, Base, and other major Layer-2 networks.&nbsp;</p><p>Building this data infrastructure from scratch would be complex and time-consuming. This is especially true given the need for scale, speed, and accuracy. According to Loomlay’s co-founder, Darius Kozlovskis, “If we wanted to do it ourselves, it would be quite a headache. We’d have to deal with indexing, node infrastructure, and data consistency across chains.”</p><h2 id="the-solution-zerion-api">The Solution: Zerion API</h2><p>Loomlay integrated <a href="https://zerion.io/api" rel="noreferrer">Zerion API</a> to solve these challenges, unlocking fast, scalable access to onchain data across all major EVM networks. Two Zerion API endpoints form the backbone of Loomlay’s agent functionality:</p><ul><li><a href="https://developers.zerion.io/reference/listwalletpositions#/" rel="noreferrer"><strong>Wallet Positions:</strong></a> Agents use Zerion’s wallet positions endpoint to instantly retrieve the latest assets held in a user’s wallet, along with valuations and token details. This data is crucial for portfolio tracking, verifying minimum balances, and tailoring agent strategies to each user.</li><li><a href="https://developers.zerion.io/reference/listwallettransactions#/" rel="noreferrer"><strong>Transaction History:</strong></a> Agents rely on Zerion’s transaction history endpoint to analyze past activity, respond to incoming transfers, and automate actions such as copy trading.&nbsp;</li></ul><blockquote>"We’re using Zerion API for both portfolio and transaction data. And it’s surprisingly fast. For example, we require at least $5 of balance to start an agent. Zerion provides this check in seconds,” says Darius. “Otherwise, we’d have to build all the infrastructure ourselves. With Zerion, it just works."</blockquote><p>Loomlay also highlighted that Zerion’s API recently shipped features that were directly on their wishlist—like real-time wallet notifications—eliminating the need for additional custom infrastructure. Loomlay is planning to use Zerion’s real-time webhooks for wallet notifications to enable agents to react to onchain events as they happen.</p><h2 id="what%E2%80%99s-next-for-loomlay">What’s Next for Loomlay</h2><p>With Loomlay’s trading agent store now in private beta, the team is doubling down on agentic trading, copy trading, and unlocking new ways for users to interact with DeFi. Upcoming features include:</p><ul><li>Integrating <a href="https://zerion.io/blog/how-to-create-real-time-ethereum-transaction-notifications-with-zerion-api/" rel="noreferrer">real-time wallet notifications</a> to enable instant copy trading and automated strategies that respond to live events</li><li>Expanding support for additional chains like Solana, making cross-chain agent automation seamless</li><li>Leveraging more onchain data sources to enrich agent intelligence and open up even more use cases</li></ul><p>Loomlay’s journey shows how Zerion API can accelerate product development and open up new possibilities in crypto automation. As Darius puts it: “We’re a happy customer. Zerion API lets us focus on building powerful agents, not maintaining infrastructure.”</p><h2 id="conclusion">Conclusion</h2><p>By leveraging Zerion API, Loomlay cut out months of backend engineering, shipped advanced agent features faster, and can now focus on what matters.&nbsp;</p><p>If you’re building AI agents, DeFi dashboards, or any crypto-powered app that needs onchain data, try Zerion API. <a href="https://dashboard.zerion.io/?utm_source=blog&amp;utm_medium=article&amp;utm_campaign=loomlay-case" rel="noreferrer">Create your free API key</a> today and unlock a new level of speed, scale, and flexibility.</p>
