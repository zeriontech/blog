<p><a href="https://www.opendue.com/"><u>Due</u></a> is building a global payments orchestration layer, enabling users to move money seamlessly across borders, stablecoins, and chains. With presence in Latin America, Africa, and Europe, Due needed a reliable way to reconcile transactions and backfill historical onchain data. While they run their own RPC nodes, they also wanted to have a single source of truth for transactions across all chains.</p><p>That’s where <a href="https://zerion.io/api" rel="noreferrer">Zerion API</a> came in.</p><p>The Challenge: Multichain Data, Without Extra Overhead</p><p>Due is a borderless payments platform that lets anyone <a href="https://www.opendue.com/accounts"><u>open a virtual account</u></a> (USD, EUR, GBP, AED, and more) in minutes and send or receive money across 80+ markets.&nbsp;</p><p>Available on web and mobile, Due enables users to add funds, make international transfers, and save up to 80% on every transaction compared to traditional rails. Built on a fully non-custodial model, Due ensures users always retain full control over their assets with no intermediaries involved.</p><figure class="kg-card kg-image-card kg-card-hascaption"><img src="https://zerion.io/blog/content/images/2025/09/-_---2-.png" class="kg-image" alt="Global payments in Due" loading="lazy" width="1440" height="900" srcset="https://zerion.io/blog/content/images/size/w600/2025/09/-_---2-.png 600w, https://zerion.io/blog/content/images/size/w1000/2025/09/-_---2-.png 1000w, https://zerion.io/blog/content/images/2025/09/-_---2-.png 1440w" sizes="(min-width: 720px) 720px"><figcaption><span style="white-space: pre-wrap;">Global payments in Due</span></figcaption></figure><p>When a new wallet is connected to Due, it starts indexing all transactions and all tokens on all its supported chains. To do this, Due already manages its own infrastructure for live transaction indexing through several RPC providers.&nbsp;</p><p>But users also wanted to see older transactions that were done before the address was connected to Due’s products.&nbsp;</p><p>When it comes to <strong>historical data and reconciliation</strong>, scaling Due’s indexing system across all supported chains quickly becomes unsustainable.</p><ul><li><strong>Backfilling gaps:</strong> When adding new tokens, contracts, or customers, Due needed a way to backfill historical transactions without indexing entire chains from scratch.</li><li><strong>Expanding coverage:</strong> With support for Polygon, Arbitrum, Base, Optimism, Ethereum, Avalanche, Starknet, and more chains in the pipeline, infrastructure complexity was only growing.</li><li><strong>Operational visibility:</strong> Teams needed a “source of truth” to verify transactions across multiple networks without juggling individual block explorers.</li></ul><p>That’s when the Due team thought of Zerion, which they’ve used a lot in their OTC trading operations.&nbsp;</p><p>As Rob Sargsian, Co-founder at Due, put it:</p><blockquote>“We’ve historically used Zerion as a meta scanner. Instead of going to PolygonScan, Arbiscan, or others, you just go to Zerion web app, and you have all of the transfers there. It’s our proxy to a source of truth for onchain transactions.”</blockquote><h2 id="the-solution-zerion-api">The Solution: Zerion API</h2><p>Zerion API became the go-to data backbone<strong> </strong>for Due’s reconciliation and backfill needs.</p><p>Based on the same data that powers its portfolio tracker and wallet, Zerion API offers historical transactions and portfolio data for millions of indexed addresses. Unlike raw block contents that RPC calls provide, Zerion offers fully interpreted data with explicit financial meaning.&nbsp;</p><p>Zerion API offered several advantages:&nbsp;</p><ul><li><strong>Seamless backfill:</strong> When new customers onboard or new tokens are added, Due uses Zerion API to quickly pull the complete transaction history for each address.</li><li><strong>Wide chain coverage:</strong> With all major EVMs and now Solana supported, Zerion API provides one of the broadest datasets in Web3.</li><li><strong>Cost-effective alternative to RPCs:</strong> Instead of maintaining multiple RPC integrations, Due can rely on Zerion API for data normalization across networks.</li><li><strong>Simple integration:</strong> Integration was completed in just a few days. Rob shared:“It’s a very simple API, easy to use. For our scope, we could probably have done it in two or three days.”</li></ul><h2 id="results">Results</h2><p>With Zerion API, Due successfully backfilled transactions in all its products.&nbsp;&nbsp;</p><ul><li><strong>Reduced infrastructure overhead</strong>: No need to maintain full historical indexing across every chain.</li><li><strong>Time saved on integration:</strong> The backfilling was done in just a few days without any blockchain engineering resources.&nbsp;&nbsp;</li><li><strong>Future-ready expansion</strong>: With upcoming support for Solana and TON, Due can extend its product offering without infrastructure bottlenecks.</li></ul><h2 id="try-zerion-api-today">Try Zerion API Today</h2><p>Zerion API makes it simple to access normalized multichain transaction and portfolio data without running your own infrastructure.
