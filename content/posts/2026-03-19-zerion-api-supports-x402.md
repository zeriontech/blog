<p><a href="https://zerion.io/api" rel="noreferrer">Zerion API</a> integrated the x402 protocol. Any AI agent with a crypto wallet can now make an API call, pay 0.01 USDC on Base, and get back structured wallet data: portfolio balances, DeFi positions, token prices, PnL, and more. It's the first step toward an API economy where agents pay for data the same way humans pay for software.</p><p>The x402 protocol brings the long-dormant HTTP 402 (Payment Required) status code to life. When a client makes an API request, the server responds with payment instructions. The client sends an onchain payment on Base, the server verifies it, and the data comes back.</p><p>Zerion used the Coinbase Developer Platform to start accepting <a href="https://www.x402.org/" rel="noreferrer">x402</a> payments for its data. Payments are done in USDC on Base, an Ethereum Layer-2 network incubated by Coinbase.</p><p>For Zerion API, this supplements the traditional human-centered onboarding flow (signup, API key, billing setup) with a single AI-native step: pay and receive data.</p><blockquote><em>"Most API providers assume there's a human signing up with a credit card. That doesn't work when your user is an AI agent. x402 lets us meet agents where they are. They pay and immediately get a response back with the data. Same portfolio, positions, and PnL data that powers the Zerion app, is now available to any agent on Base for a cent per call." </em>— Abi Dharshan, Product Lead, Zerion API</blockquote><blockquote><em>"Agents need to spend money, and right now that's harder than it should be. x402 on Base gives agents a way to pay for services like Zerion's API without a human in the loop. In turn, Zerion API gives agents a way to take part in the onchain economy. They can use this API to watch their own trading wallets, generate value-added analysis, or provide other services." </em>— Kevin Leffew, GTM Lead, Coinbase Developer Platform</blockquote><h2 id="why-zerion-data">Why Zerion data</h2><p>Raw blockchain data is hard to use. A token transfer on Ethereum looks nothing like a Solana swap. Neither comes with human-readable labels, USD values, or protocol context. Building that normalization layer from scratch takes months.</p><p>Zerion API returns interpreted, normalized data. It’s the same data that powers the Zerion portfolio tracker app. Token names, prices, and other metadata are included in every response. This makes the data directly usable in dashboards, reports, and agent workflows without additional processing.</p><p>For AI agents, this matters because LLMs work best with structured data. An agent that receives a clean JSON response with token symbols, USD values, and position types can immediately reason about it. An agent that receives raw calldata cannot.</p><h2 id="built-for-autonomous-agents">Built for autonomous agents</h2><p>Traditional API access models assume a human on the other end. Someone creates an account, enters a credit card, and manages a subscription. AI agents can't do that.&nbsp;</p><p>x402 enables a better agent-first payment experience., An agent discovers a Zerion endpoint, pays from its own wallet, and receives structured data. The flat pricing gives agents a clear budget constraint, so they can reason about the marginal cost before making requests.</p><p>Agents need structured wallet state, portfolio truth, and reliable pricing to act autonomously. With x402 access to Zerion's data, an agent can check its own portfolio, track PnL on positions it holds, and layer Zerion queries into more complex workflows.</p><h2 id="already-powering-ai-agents-at-scale">Already powering AI agents at scale</h2><p>Zerion API already powers leading <a href="https://zerion.io/blog/tag/ai-agents/" rel="noreferrer">AI agents</a>, which collectively making millions of API calls. x402 support extends this access to any agent with a wallet, no contract or integration required.</p><h2 id="available-data">Available data</h2><p><a href="http://zerion.io/agents" rel="noreferrer">Zerion for agents</a> offers the same onchain wallet data available in Zerion:</p><ul><li>Wallet portfolio and token balances across chains</li><li>DeFi positions (staking, lending, LP, and more)</li><li>Token prices and historical charts</li><li>PnL for any token</li><li>Transaction history</li></ul><h2 id="how-to-get-started">How to get started</h2><h3 id="fast-path-try-it-in-10-seconds">Fast path: try it in 10 seconds</h3><p>You can test x402 access with any wallet that supports Base. No setup required: just call a pay-gated endpoint and see the response.</p><p>Try this live endpoint to fetch all transactions for vitalik.eth: <a href="https://api.zerion.io/v1/wallets/0xd8da6bf26964af9d7eed9e03e53415d37aa96045/transactions/"><u>api.zerion.io/v1/wallets/0xd8da.../transactions/</u></a></p><h3 id="production-path-set-up-your-ai-agent">Production path: set up your AI agent</h3><p>This setup gives your agent a wallet and teaches it how to call Zerion:</p><ol><li><strong>Give your agent a wallet. </strong>Install Coinbase Agentic Wallet or CDP Server Wallet to create a crypto wallet on Base.</li><li><strong>Add the Zerion API CLI.</strong> This gives your agent structured descriptions of available endpoints and data types, so it knows what to call and when.&nbsp;</li><li><strong>Prompt.</strong> Ask the agent to fetch portfolio for any wallet address. The agent will pay $0.01 per request via x402 on Base and receive structured JSON responses.</li></ol><p>For full instructions and a complete list of available endpoints, visit the dedicated <a href="https://github.com/zeriontech/zerion-ai"><u>GitHub repository for Zerion API with AI</u></a>.</p><h2 id="part-of-zerions-agent-infrastructure">Part of Zerion's agent infrastructure</h2><p>x402 support is one piece of a broader effort to make Zerion's data accessible to autonomous systems. Three layers are coming together:</p><ul><li><strong>x402 payments</strong> — permissionless, pay-per-request access (live now)</li><li><a href="https://github.com/zeriontech/zerion-ai/tree/main/cli" rel="noreferrer"><strong>Zerion CLI</strong></a> — structured access to Zerion endpoints that agents can discover and use</li><li><a href="https://developers.zerion.io/reference/building-with-ai" rel="noreferrer"><strong>MCP server</strong></a> — a standardized way for AI to access Zerion API data</li></ul><p>Together, these let agents discover, understand, and pay for blockchain data without human setup.</p>
